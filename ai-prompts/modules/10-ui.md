## Your Task

Implement both CLI console and web dashboard interfaces for managing the RMM system.

---

## Step 1: Create Start-Console.ps1 - CLI Dashboard

Create `RMM/scripts/ui/Start-Console.ps1`:

**Purpose:** Interactive command-line interface for RMM management

**Console Layout:**

```text
+======================================================================+
|                    myTech.Today RMM Console v2.0                     |
+======================================================================+
|  Fleet Status: 147/150 Online (98%)     Active Alerts: 3 (1 Critical)|
+======================================================================+
|  [1] Device Management     [5] Update Management                     |
|  [2] Real-Time Monitoring  [6] Remote Actions                        |
|  [3] Alerts Dashboard      [7] Reports                               |
|  [4] Inventory Browser     [8] Settings                              |
|                            [0] Exit                                  |
+======================================================================+
Select option: _
```

**Console Features:**
- Device list with status indicators
- Real-time metric display (auto-refreshing)
- Alert acknowledgement and resolution
- Quick action execution
- Keyboard navigation
- Color-coded status (Green=Healthy, Yellow=Warning, Red=Critical)
- Search and filter devices
- Drill-down to device details

**Functions to Implement:**
1. `Show-RMMConsoleMenu` - Display main menu
2. `Show-RMMDeviceList` - List devices with status
3. `Show-RMMAlertDashboard` - Display active alerts
4. `Show-RMMDeviceDetails` - Show detailed device info
5. `Invoke-RMMConsoleAction` - Execute actions from console

---

## Step 2: Create Start-WebDashboard.ps1 - Web Interface

Create `RMM/scripts/ui/Start-WebDashboard.ps1`:

**Purpose:** Self-hosted web dashboard for RMM management

**Technology:**
- Self-hosted HTTP listener (no IIS required)
- Port 8080 (configurable)
- PowerShell HttpListener class

**Web Dashboard Pages:**
- `/` - Overview dashboard with fleet status
- `/devices` - Device list with filtering
- `/devices/{id}` - Individual device details
- `/alerts` - Alert management
- `/reports` - Report viewer and generator
- `/actions` - Action execution interface
- `/settings` - Configuration management

**Features:**
- Responsive HTML5 dashboard
- Real-time updates via polling or Server-Sent Events
- Device drill-down views
- Alert management (acknowledge, resolve)
- Action execution
- Report viewing and generation
- Mobile-friendly design

---

## Step 3: Create Web Assets

Create the following files in `RMM/scripts/ui/web/`:

### index.html
- Main dashboard page
- Fleet status overview
- Active alerts summary
- Device status chart
- Recent actions list

### styles.css
- Responsive CSS styling
- Color scheme matching myTech.Today branding
- Mobile-friendly layout
- Status color coding

### app.js
- JavaScript for interactivity
- AJAX calls to PowerShell backend
- Auto-refresh functionality
- Chart rendering (using Chart.js or similar)
- Form handling for actions

**Technology Stack:**
- HTML5 + CSS3 (no heavy frameworks)
- Vanilla JavaScript or minimal library
- PSWriteHTML module for report generation
- Built-in PowerShell HTTP listener
